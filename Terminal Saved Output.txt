  How it works

  1. Intro screen - Visitor sees a calming message: "Pause. Breathe. Let a single image hold your attention for sixty seconds."
  2. Click "Begin" - A random photo from your collection fades in, filling the screen
  3. Timer appears - A subtle circular progress ring in the bottom-right corner counts down from 1:00. The header fades away to minimize distraction.
  4. Completion - After 60 seconds, the message "Thank you for being present" appears with options to view another random photo or explore collections

  Features

  - Keyboard support: Press Space to start or restart
  - Header auto-hides during viewing, reappears after
  - Smooth transitions throughout
  - Mobile responsive

  In moment.html, update the photos array (around line 60) with your actual image paths:

  const photos = [

⏺ There are a few approaches depending on how you plan to host the site:

  1. Local Images (Simplest)
  Create an images folder structure matching your collections:
  photography-portfolio/
  ├── images/
  │   ├── hero.jpg
  │   ├── urban/
  │   │   ├── 01.jpg
  │   │   ├── 02.jpg
  │   │   └── ...
  │   ├── portraits/
  │   ├── nature/
  │   └── abstract/
  Then update the src attributes from https://picsum.photos/seed/... to images/urban/01.jpg (or ../../images/urban/01.jpg from collection pages).

  2. Cloud/CDN Hosting
  Services like Cloudinary, ImageKit, or Imgix provide:
  - Automatic image optimization and resizing
  - Fast global delivery
  - You'd replace URLs with their hosted links

  3. Your Web Host
  If deploying to Netlify, Vercel, or traditional hosting, the local images approach works - they get uploaded with your site files.

  ---
  Recommendation: Start with the local images/ folder approach. It's straightforward and works with any hosting. For a photography portfolio, I'd suggest:
  - Keep originals at reasonable web sizes (1600-2400px on the long edge)
  - Use JPG for photos, optimize for web (80-85% quality)
  - Name files descriptively or numbered for easy management
